A Case For Pure HTML

Kresten Jacobsen, 2019-08-24

There's like a gazillion fancy webframeworks, cool blog systems, static site generators, so why old timey HTML?

My first and most important argument for html, is durability. If I find the source for this site in 20 years odds are that I'll still be able to view it immidiately in a browser with minimal work. I've tried doing the same with different kinds of frameworks and static site generators but they tend to change over time and may not be able to render the source files anymore.

The second reason is portability. HTML kan be pushed to more or less every webserver in the world. As such nothing hinges on a specific setup and it is possible to move without notice.

HTML is secure. HTML is in and of itself as secure as a text document. Because it is a text document. Where other languages relies on a webserver keeping secrets (e.g. password to a database as with PHP & Wordpress) HTML doesn't need any of that.

The fourth good reason for choosing to write in HTML is that it's simple. HTML is pretty straight forward - and if you want to you can read the full source for this site and edit it offline. I can use any editor I want on any computer. When I write a new post I just copy the old one and delete the content (I did that anyway with the static site generators).

Oh, and it's fast. Of course while you certainly can write slow sites by hand at least your in change. This site should load in less than a secon. If not something have gone wrong.
